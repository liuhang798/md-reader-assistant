# Security Policy

## Supported version

Security fixes are provided for the latest published release.

## Reporting a vulnerability

Please do not publish sensitive vulnerability details in a public issue. Email `lh805798@163.com` with:

- the affected version and operating system;
- clear reproduction steps;
- the potential impact;
- any suggested mitigation.

Do not attach private Markdown documents or credentials. A response will be provided as soon as practical.
